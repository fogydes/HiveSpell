# Security Policy

## Reporting A Vulnerability

If you discover a security vulnerability in HiveSpell, please report it privately.

Please include:

- a clear description of the issue
- steps to reproduce it
- impact assessment if known
- screenshots, logs, or proof of concept details when useful

## Please Do Not

- open a public issue for an unpatched security vulnerability
- publish exploit details before maintainers have had time to investigate
- include secrets, tokens, or personal data in public reports

## Response Goals

Maintainers should aim to:

- acknowledge reports promptly
- investigate and confirm impact
- communicate remediation steps or timelines when possible
- credit responsible disclosure where appropriate

## Scope

Security concerns may include:

- authentication and authorization flaws
- insecure database access or privilege escalation
- storage or file upload issues
- account takeover paths
- client-side behaviors that expose sensitive data

## Supported Versions

The project does not currently maintain multiple supported release lines. Security fixes are expected to land on the active main branch.
